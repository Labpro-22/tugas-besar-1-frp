Untuk integrasi UI dengan engine, apakah Anda setuju GameEngine mengembalikan CommandResult terstruktur, atau Anda ingin pola observer/event bus terpisah?
- Mengingat kita saat ini sedang fokus mengejar MVP (Minimum Viable Product) dalam bentuk CLI terlebih dahulu, saya lebih setuju menggunakan CommandResult Terstruktur. >
Alasannya, CommandResult jauh lebih mudah di-trace dan di-debug alurnya secara sekuensial (request-response) di dalam terminal. Pola Observer/Event Bus memang lebih OOP dan bagus untuk GUI, tapi risikonya terlalu besar (susah di-debug jika terjadi chain-reaction event) untuk fase CLI saat ini. Nanti kalau CLI sudah stabil dan kita mau lanjut ke bonus GUI (SFML/Raylib), barulah kita bisa pertimbangkan refactor ke Event Bus/Observer agar UI bisa bereaksi secara asynchronous.
